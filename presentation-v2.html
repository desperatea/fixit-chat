Backend

⚡

Python 3.12 + FastAPI

Полностью async. Автогенерация OpenAPI/Swagger. Dependency injection из коробки. Type hints обязательны.

SQLAlchemy 2.0 (async)

Alembic

Автогенерация миграций из моделей. Команда make makemigrations создаёт файл миграции, make migrate применяет. Версионность схемы БД.

🔌 Сценарий 3: Гибрид (сервер в офисе, сайт снаружи)

Сервер чата стоит в офисе, но должен быть доступен из интернета для посетителей сайта.

Варианты:

  • 1. Проброс порта на роутере (443 → сервер). Нужен статический IP или DDNS.
  • 2. Reverse SSH / Cloudflare Tunnel — сервер сам пробивает туннель наружу. Без проброса портов.
  • 3. VPS как прокси — дешёвый VPS проксирует трафик в офис через WireGuard/VPN.
# Cloudflare Tunnel (бесплатно)
cloudflared tunnel --url http://localhost:80

# WireGuard VPN (VPS ↔ офис)
# На VPS nginx проксирует:
proxy_pass http://10.0.0.1:80;
# 10.0.0.1 = офисный сервер через VPN

# Port forwarding на роутере
WAN:443 → LAN:10.66.0.192:80

Доступ для удалённых сотрудников

Метод Как работает Когда использовать
VPN Сотрудник подключается к офисному VPN, получает IP из подсети Есть VPN-инфраструктура, нужна безопасность
IP Whitelist Домашний IP сотрудника добавляется в настройках админки Статический IP дома, нет VPN

Безопасность

Эшелонированная защита

6 уровней — каждый работает независимо. Компрометация одного не открывает систему.

# Уровень Что делает Детали реализации
1 HTTPS Шифрование канала передачи TLS 1.3, wildcard *.fixitmail.ru, nginx SSL termination
2 Security Headers 16 заголовков на каждый ответ CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy
3 CORS Whitelist Только разрешённые домены Динамический список из БД, без перезапуска
4 Rate Limiting Защита от DDoS / сканирования Redis sliding window: 100 req/min API, 10 req/min login
5 Brute Force + IP Whitelist Защита админки 5 попыток → блок IP 15 мин. Whitelist из настроек
6 AES-256-GCM + bcrypt Шифрование данных в БД Уникальный nonce на каждое значение. Пароли — bcrypt. JWT — HS256

🔐 Шифрование: что и как

# Зашифрованные поля:
visitor_name, visitor_phone, visitor_org
message.content, note.content

# Алгоритм:
AES-256-GCM (authenticated encryption)
Key: 32 bytes из env ENCRYPTION_KEY
Nonce: 12 bytes, random, уникален

# Формат хранения в БД:
base64(nonce[12] + ciphertext + tag[16])

# Даже при утечке дампа БД —
# данные нечитаемы без ключа

🔑 Аутентификация: JWT + Redis

# Access Token (в памяти браузера)
Алгоритм: HS256
TTL: 15 минут
Payload: { sub: agent_id, type: access }

# Refresh Token (httpOnly cookie)
TTL: 7 дней
Хранится в Redis: refresh:{agent_id}:{suffix}

# Logout = удаление из Redis
# Токен мгновенно невалиден

# Посетители: visitor_token (hex UUID)
# Доступ только к своей сессии

⚙ Управление

make up # Запуск + пересборка
make down # Остановка
make logs # Логи backend (live)
make create-admin # Новый администратор
make migrate # Применить миграции
make makemigrations # Создать миграцию
make test # Запуск тестов
make backup # Бэкап БД
make shell # Bash в контейнере

# Перезапуск отдельного сервиса
docker compose restart backend

# Пересборка с нуля
docker compose build --no-cache widget

📜 Логирование

Structlog — JSON в stdout

Каждая запись — структурированный JSON. Легко парсить, фильтровать, агрегировать. Docker собирает через json-file driver.

{
"event": "session_created",
"session_id": "a1b2c3...",
"visitor_name": "[encrypted]",
"timestamp": "2025-01-15T10:30:00Z",
"level": "info"
}

Просмотр логов

Все контейнеры логируют в stdout — Docker Compose агрегирует.

# Все контейнеры
docker compose logs -f

# Только backend
make logs

# Фильтр по уровню
docker compose logs backend | grep "error"

# Последние 100 строк nginx
docker compose logs --tail=100 nginx

📈 Мониторинг (Prometheus + Grafana)

Архитектура готова к подключению Prometheus + Grafana. Эндпоинт /metrics уже в маршрутизации, доступен по IP whitelist.

Планируемые метрики:

chat_sessions_total chat_sessions_active chat_messages_total chat_response_time chat_ratings ws_connections_active http_requests_total http_request_duration

Также можно подключить:

  • 📊 Grafana дашборд — визуализация метрик, алерты
  • 📩 Alertmanager — уведомления в Telegram при проблемах
  • 📋 Loki — агрегация логов из всех контейнеров
  • 💡 Uptime Kuma — мониторинг доступности /health
